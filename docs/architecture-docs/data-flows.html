Data Flows

Visual walkthrough of the 4 core processing pipelines

1. Conversation Pipeline

The main message-handling path in ConversationPipeline.handleConversation(). A Telegram message enters the pipeline, passes through auth and session checks, triggers a parallel RAG pre-fetch, builds the system prompt, calls the LLM with optional tool execution rounds, persists the exchange to the session, and sends the reply back to Telegram.

Conversation Pipeline Flow

2. Agent Loop

The autonomous execution loop in AgentLoop. Each bot runs in its own independent async loop (runBotLoop). The loop supports two modes: periodic (timer-based with configurable intervals) and continuous (runs back-to-back with a short pause). Every N cycles the Strategist phase fires to review goals and adjust focus before the Planner generates a step list for the Executor.

Agent Loop Flow

3. Collaboration Flow

CollaborationManager supports three collaboration modes between bots. Visible collaboration sends real Telegram messages between bots with multi-turn discussion support. Internal collaboration uses collaborationStep() for direct LLM-to-LLM communication with session tracking. Delegation hands off a task to another bot's full LLM pipeline (without tools to prevent loops).

Collaboration Flow — 3 Modes

4. Memory Search Flow

The hybrid search in memory/search.ts combines three strategies: vector search using cosine similarity on embeddings, keyword search via SQLite FTS5 with BM25 scoring, and Core Memory search with importance weighting. Results from all three are merged, deduplicated, scored with configurable weights (vectorWeight / keywordWeight), and returned as the top-K matches.
