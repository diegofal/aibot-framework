Configuration

Config structure, Zod schemas, and per-bot overrides

Config File

The entire framework is configured through a single JSON file located at config/config.json. It is loaded at startup by the loadConfig() function in src/config.ts, which:

  • Reads the JSON file from disk using readFileSync
  • Substitutes environment variables using ${VAR_NAME} syntax
  • Validates the entire structure against a root ConfigSchema (Zod)
  • Returns a fully typed Config object with all defaults applied

If validation fails, Zod issues are logged with their path and message before throwing an Invalid configuration error. This fail-fast approach ensures no bot starts with a malformed config. 

// Loading config at startup
import { loadConfig } from './config';

const config = await loadConfig('./config/config.json');

Per-Bot Overrides

The function resolveAgentConfig() in src/config.ts merges global defaults with per-bot overrides to produce a flat ResolvedAgentConfig. This pattern allows most settings to be defined once at the top level while individual bots override only what they need.

ResolvedAgentConfig Interface

export interface ResolvedAgentConfig {
  model: string;
  llmBackend?: 'ollama' | 'claude-cli';
  soulDir: string;
  workDir: string;
  systemPrompt: string;
  temperature: number;
  maxHistory: number;
}

Merge Rules

Each field is resolved by checking the per-bot config first, then falling back to the global default. The table below shows the exact resolution chain:

Resolved Field Per-Bot Source Global Fallback
model botConfig.model globalConfig.ollama.models.primary
llmBackend botConfig.llmBackend undefined (defaults to ollama)
soulDir botConfig.soulDir {globalConfig.soul.dir}/{botConfig.id}
workDir botConfig.workDir {globalConfig.productions.baseDir}/{botConfig.id}
systemPrompt botConfig.conversation.systemPrompt globalConfig.conversation.systemPrompt
temperature botConfig.conversation.temperature globalConfig.conversation.temperature
maxHistory botConfig.conversation.maxHistory globalConfig.conversation.maxHistory

Resolution implementation

export function resolveAgentConfig(
  globalConfig: Config,
  botConfig: BotConfig
): ResolvedAgentConfig {
  return {
    model:        botConfig.model
                    ?? globalConfig.ollama.models.primary,
    llmBackend:   botConfig.llmBackend,
    soulDir:      botConfig.soulDir
                    ?? `${globalConfig.soul.dir}/${botConfig.id}`,
    workDir:      botConfig.workDir
                    ?? `${globalConfig.productions?.baseDir ?? './productions'}/${botConfig.id}`,
    systemPrompt: botConfig.conversation?.systemPrompt
                    ?? globalConfig.conversation.systemPrompt,
    temperature:  botConfig.conversation?.temperature
                    ?? globalConfig.conversation.temperature,
    maxHistory:   botConfig.conversation?.maxHistory
                    ?? globalConfig.conversation.maxHistory,
  };
}

Environment Variables

The config file supports environment variable substitution using the ${VAR_NAME} syntax. Before Zod validation, the substituteEnvVars() function recursively walks the parsed JSON and replaces every ${...} occurrence with the corresponding process.env value.

If a referenced environment variable is not defined, an error is thrown immediately with a clear message: Environment variable VAR_NAME is not defined.

Common usage patterns

{
  "bots": [
    {
      "id": "asistente",
      "name": "Asistente",
      "token": "${BOT_ASISTENTE_TOKEN}"
    }
  ],
  "ollama": {
    "baseUrl": "${OLLAMA_URL}"
  },
  "webTools": {
    "search": {
      "apiKey": "${SEARCH_API_KEY}"
    }
  },
  "phoneCall": {
    "accountSid": "${TWILIO_SID}",
    "authToken": "${TWILIO_AUTH_TOKEN}",
    "fromNumber": "${TWILIO_FROM_NUMBER}"
  }
}

Substitution rules

Rule Behavior
String values ${VAR} patterns are replaced inline. Multiple substitutions per string are supported.
Arrays Each element is recursively processed
Objects Each value is recursively processed (keys are not substituted)
Non-strings Numbers, booleans, null — passed through unchanged
Missing variable Throws Error immediately (fail-fast)

Helper functions

In addition to loadConfig() and resolveAgentConfig(), the config module exports one more utility:

getSkillConfig<T>(config, skillId)

Returns the per-skill configuration from config.skills.config[skillId], falling back to an empty object {} if not present. The generic parameter T allows callers to cast the result to their expected shape without additional validation.
