Operator API

The Operator API provides programmatic control over bots without requiring a chat channel (Telegram, WhatsApp, etc.). It enables triggering skill commands, inspecting bot capabilities, and automating operations via HTTP.

Authentication

All Operator API endpoints live under /api/agents/ and are protected by the existing tenant auth middleware. Include your API key in the Authorization header: 

Authorization: Bearer YOUR_API_KEY

Endpoints

List Bot Skills & Commands

GET /api/agents/:botId/skills

Returns all loaded skills for a bot, including available commands.

Response

{
  "botId": "default",
  "skills": [
    {
      "id": "reflection",
      "name": "Reflection",
      "version": "1.0.0",
      "description": "Self-reflection on recent conversations",
      "commands": [
        { "name": "reflect", "description": "Run self-reflection on recent conversations" }
      ]
    }
  ]
}

Execute Skill Command

POST /api/agents/:botId/skills/:skillId/:command

Executes a skill command headlessly. The bot must be running and have the skill enabled.

Request Body (optional)

{
  "args": ["arg1", "arg2"]
}

Success Response (200)

{
  "ok": true,
  "result": "Reflection complete. Updated IDENTITY.md and journal.",
  "durationMs": 12345
}

Error Response (404 / 500)

{
  "error": "Bot default is not running",
  "durationMs": 2
}

Examples

Trigger reflection manually

curl -X POST http://localhost:3000/api/agents/default/skills/reflection/reflect \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json"

List available commands

curl http://localhost:3000/api/agents/default/skills \
  -H "Authorization: Bearer $API_KEY"

Execute with arguments

curl -X POST http://localhost:3000/api/agents/default/skills/daily-briefing/brief \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"args": ["--force"]}'
