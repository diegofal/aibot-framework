AI Coach / Student

Personalized coaching assistants with per-user goal tracking, topic enforcement, and proactive check-ins. The primary use case driving the current feature set.

Key Features Used

  • Per-user goals — manage_goals tool with scope:"user" stores goals in memory/users/{userId}/GOALS.md. Each student has independent objectives.
  • Topic guard — Keeps conversations coaching-focused. Configure botPurpose, allowedTopics, and strictness to prevent off-topic drift.
  • Proactive messaging — send_proactive_message tool enables the agent loop to send follow-ups, reminders, and check-ins without user initiation.
  • User awareness in agent loop — agentLoop.userAwareness injects active users summary into planner prompts so the bot knows who to reach out to.
  • Identity verification — HMAC-based userHash for widget embedding prevents senderId spoofing.
  • Chat history persistence — Widget persists chatId/senderId in localStorage; history loads on reconnect.

Configuration Example

{
  "topicGuard": {
    "enabled": true,
    "botPurpose": "Sales coaching for B2B teams",
    "allowedTopics": ["sales", "negotiation", "pipeline", "objection handling"],
    "strictness": "moderate",
    "failOpen": true
  },
  "userIsolation": { "enabled": true },
  "agentLoop": {
    "enabled": true,
    "mode": "periodic",
    "userAwareness": { "enabled": true, "activeWindowHours": 24, "maxUsers": 5 }
  }
}

Variants

AI Tutor (language learning, spaced repetition), AI Mentor (career coaching), AI Fitness Coach (workout tracking with per-user goals), AI Therapist Companion (strict topic guard + crisis detection).

AI Customer Support Agent

Product-focused support with per-user context, topic guardrails, and REST API integration with helpdesk tools.

Key Features Used

  • Topic guard — On-product only. Blocks off-topic requests with a friendly redirect.
  • REST API — POST /api/v1/chat/:botId integrates with existing helpdesk systems (Zendesk, Intercom) as a backend service.
  • Per-user memory — Remembers context across tickets without leaking between customers.
  • Escalation — ask_human tool for handoff to human agents when confidence is low.
  • Widget embedding — Drop-in chat widget for product websites with identity verification.

Configuration Example

{
  "topicGuard": {
    "enabled": true,
    "botPurpose": "Customer support for Acme SaaS product",
    "allowedTopics": ["billing", "account", "features", "bugs", "integrations"],
    "blockedTopics": ["competitors", "investment advice"],
    "strictness": "strict",
    "customRejectMessage": "I can only help with Acme product questions. For other topics, please contact sales@acme.com."
  },
  "userIsolation": { "enabled": true },
  "userIdentityVerification": { "enabled": true, "required": true }
}

What’s Missing

Ticket/thread mapping to external systems, SLA tracking, canned response library. Escalation via ask_human covers basic handoff.

AI Sales Assistant (SDR)

Deal tracking, follow-up automation, and pipeline coaching. The agent loop proactively reaches out to leads.

Key Features Used

  • Per-user goals — Track deal stages per lead (e.g., “Discovery”, “Proposal Sent”, “Negotiation”).
  • Proactive messaging — Automated follow-ups and check-ins via send_proactive_message.
  • Topic guard — Sales-focused conversations only.
  • REST API — Integrate with CRM systems for lead data and deal updates.
  • Agent loop user awareness — Planner sees active leads and decides who needs follow-up.

What’s Missing

Native CRM integration tool (Salesforce, HubSpot), lead scoring model, email channel.

AI Onboarding Buddy

Employee or user onboarding with checklist tracking and nudges. Guides new users through setup steps.

Key Features Used

  • Per-user goals — Onboarding checklist as goals (e.g., “Complete profile”, “Watch intro video”, “Set up integrations”).
  • Proactive messaging — Nudges for incomplete steps.
  • Identity verification — Employee ID verification for internal tools.
  • Widget embedding — Embedded in the product for in-app guidance.

What’s Missing

Progress tracking dashboard, completion webhook to external systems, structured step sequencing.

Internal Knowledge Bot

Company knowledge base with identity-verified access and embeddable widget. Answers questions from internal docs.

Key Features Used

  • REST API — Embed in Slack/Teams via webhook relay.
  • Identity verification — SSO-backed HMAC for authenticated access.
  • Topic guard — Company-relevant topics only.
  • RAG — Semantic search over ingested documents via the memory system.
  • Chat history — Persistent conversations for reference.

What’s Missing

SSO/SAML integration, permission-scoped RAG (different access levels per user), bulk document ingestion pipeline.

White-Label Bot Platform (BaaS)

Multi-tenant platform where each tenant creates, customizes, and embeds bots. The full BaaS stack.

Key Features Used

  • Tenant isolation — Data, config, billing, and collaboration fully scoped per tenant.
  • Templates — Reusable bot templates that tenants instantiate and customize.
  • Customization overlays — Identity, knowledge, goals, rules, topic guard, branding — all per-tenant.
  • Widget embedding — Each tenant embeds widgets on their own sites with their branding.
  • API keys — Programmatic access for tenant backend integrations.
  • Billing & quota — Per-plan limits, usage metering, Stripe integration.

Platform Variants

  • Bot marketplace — Templates + customization layer = tenants pick a template, customize soul/goals/topic guard.
  • Agency model — One agency manages multiple tenant bots, each with separate billing and user isolation.

Recommended approach for validating use case scenarios.

Identity Verification (Widget)

# Generate HMAC hash for a user
echo -n "user-123" | openssl dgst -sha256 -hmac "idsec_YOUR_SECRET"

# Widget: add data-user-hash="<hash>" to script tag
# Invalid hash → WebSocket connection rejected
# No hash + required:false → anonymous access allowed

REST Chat API

# Single-tenant
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -H 'Content-Type: application/json' \
  -d '{"message":"Hello","senderId":"user-123"}'

# Multi-tenant
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -H 'Authorization: Bearer TENANT_API_KEY' \
  -d '{"message":"Hello","senderId":"user-123","userHash":"HMAC_HASH"}'

Topic Guard

# On-topic → normal response
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -d '{"message":"How do I handle sales objections?","senderId":"u1"}'

# Off-topic → rejection message
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -d '{"message":"Recipe for pasta carbonara?","senderId":"u1"}'

Per-User Goals

# User A sets a goal
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -d '{"message":"Add a goal: master cold calling","senderId":"user-A"}'

# User B sees only their own goals
curl -X POST http://localhost:3000/api/v1/chat/BOT_ID \
  -d '{"message":"List my goals","senderId":"user-B"}'

# Verify isolation: data/bots/BOT_ID/memory/users/user-A/GOALS.md
#                   data/bots/BOT_ID/memory/users/user-B/GOALS.md

Chat History

# Fetch history for a chat session
curl "http://localhost:3000/api/v1/chat/BOT_ID/history?chatId=CHAT_ID&senderId=user-123&limit=50"
