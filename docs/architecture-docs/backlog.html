Audio Input (Voice-to-Text)

Code Complete
processVoice() in src/media.ts sends OGG to Whisper endpoint. Handler message:voice in src/bot/media-handlers.ts injects transcription into conversation. WhisperConfigSchema supports OpenAI and local whisper.cpp endpoints. Authorization header sent conditionally when apiKey is present. 13 unit tests in tests/media.test.ts.
Remaining
  • Manual testing with real Telegram voice notes (requires OPENAI_API_KEY)
  • "Transcribing..." feedback message while processing
  • Support for message:audio (regular audio files, not just voice notes)

Audio Output (TTS)

Code Complete
generateSpeech() in src/tts.ts calls ElevenLabs API, returns OGG/Opus. Inbound mode: TTS activates when user sends voice. Integrated in conversation-pipeline.ts. Markdown stripping, text truncation, typing indicator (record_voice). 22 unit tests + 4 integration tests.
Remaining
  • Manual testing with real voice notes (requires ELEVENLABS_API_KEY)
  • Multi-provider support (OpenAI TTS as cheaper alternative)
  • Response caching for frequent answers
  • Outbound mode: send_voice_message tool where LLM decides when to speak
  • message:audio support (regular audio files don’t trigger TTS)

WhatsApp Integration

Research
The bot is currently Telegram-only via grammy. Minimal platform abstraction exists (TelegramClient interface in src/core/types.ts).
Integration Options
OptionApproachTrade-offs
A WhatsApp Business Cloud API (Meta) Free for user-initiated messages (24h window). Requires Facebook Business account. Template approval for outbound.
B Via provider (Twilio, MessageBird) Friendlier API. Twilio already partially integrated (phone calls). Additional cost per message.
C Open-source (whatsapp-web.js, Baileys) Free but high risk of ban. Unstable reverse-engineered protocol. Not recommended for production.
Required Refactoring
  • Abstract TelegramClient to MessagingPlatform interface
  • Handler adapter layer (grammy handlers → platform-agnostic)
  • Media handling (WhatsApp has its own media ID / download system)
  • Session key adaptation for WhatsApp chat IDs
  • Skills using Telegram-specific features (inline keyboards, callbacks)
Next Steps
  • Decide: refactor for multi-channel vs. separate WhatsApp bot sharing core
  • Design MessagingPlatform interface and adapter pattern
  • Choose WhatsApp API provider
  • POC with echo bot
