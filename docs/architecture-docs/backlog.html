▶ A2A Protocol (Agent-to-Agent) 16 stories · 2 phases Designed

Native TypeScript implementation of A2A v0.3.0 in the existing web server. Makes every bot a standards-compliant A2A agent. Enables discovery, consumption, and collaboration with external A2A agents. Builds on existing MCP infrastructure patterns in src/mcp/.

Phase 1 — A2A Server + Client (11 stories)
    ▶ Types — A2A v0.3.0 protocol type definitions

    Create src/a2a/types.ts.

    • A2aAgentCard, A2aSkill, A2aCapabilities
    • A2aMessage, A2aPart (TextPart, DataPart, FilePart)
    • A2aTask, TaskState, TaskStatus
    • MessageSendParams, MessageSendResult
    References

    Reuse JSON-RPC base from src/mcp/types.ts. See A2A spec.

    ▶ Config — schema and defaults

    Add a2a block to src/config.ts (Zod schema) and config/config.json.

    • server.enabled, server.basePath (default /a2a), server.authToken
    • clients[] — { name, url, autoConnect, authToken? }
    • directory.enabled, directory.remoteUrl, directory.heartbeatIntervalMs, directory.staleTtlMs
    ▶ Agent Card Builder — auto-generate from bot config

    Create src/a2a/agent-card-builder.ts.

    • buildAgentCard(botConfig, toolDefs, baseUrl) → A2aAgentCard
    • Group tools by TOOL_CATEGORIES into A2A skills
    • Extract description from soul system prompt
    ▶ Task Store — in-memory task lifecycle

    Create src/a2a/task-store.ts.

    • create(), get(), update(), cancel()
    • States: submitted → working → completed / failed / canceled
    • TTL-based cleanup
    ▶ Executor — headless LLM + tools message processor

    Create src/a2a/executor.ts. Processes A2A messages without Telegram or sessions.

    • Extract text from A2A Message parts
    • Build system prompt via SystemPromptBuilder
    • Get tools via ToolRegistry.getDefinitionsForBot(botId)
    • Call LLM, return A2A Message + Task status
    Reference

    src/bot/conversation-pipeline.ts — adapt without Telegram deps.

    ▶ Server — JSON-RPC HTTP handler

    Create src/a2a/server.ts.

    • Per-bot routes: GET /a2a/:botId/.well-known/agent-card.json, POST /a2a/:botId/
    • JSON-RPC: message/send, tasks/get, tasks/cancel
    • Bearer token auth, rate limiting
    Reference

    src/mcp/server.ts — same HTTP pattern.

    ▶ Web integration — mount routes, wire lifecycle
    • Mount in src/web/server.ts under config.a2a.server.basePath
    • startA2a() / stopA2a() in BotManager
    ▶ Client + Pool — consume external A2A agents

    Create src/a2a/client.ts and src/a2a/client-pool.ts.

    • A2aClient: fetchAgentCard(), sendMessage(), getTask()
    • A2aClientPool: manages connections, connectAll(), getAllAgents()
    Reference

    src/mcp/client.ts + client-pool.ts

    ▶ Tool Adapter — external agents as framework tools

    Create src/a2a/tool-adapter.ts.

    • adaptA2aAgent(agentCard, client) → Tool[]
    • Naming: a2a_<agentName>_<skillId>
    • ToolRegistry.registerA2aTools()
    Reference

    src/mcp/tool-adapter.ts

    ▶ Collaboration — A2A as transport in CollaborationManager
    • Route to A2aClientPool when target is a2a-external
    • Register A2A agents in AgentRegistry
    • isTargetAvailable() checks MCP + A2A
    Reference

    src/bot/collaboration.ts — mcpCollaborationStep()

    ▶ Tests — unit tests for all Phase 1 modules
    • Types: serialization / deserialization
    • Agent Card Builder: valid cards from real config
    • Task Store: lifecycle, TTL, concurrency
    • Executor: message processing, tool calls, errors
    • Server: JSON-RPC dispatch, auth, rate limiting
    • Client: fetch, send, timeout, errors
    • Tool Adapter: naming, execution, result mapping

▶ Multi-Channel Messaging 2 stories Research

Abstract the Telegram-specific transport layer into a MessagingPlatform interface to support WhatsApp, Discord, and future channels without duplicating core logic.

    ▶ WhatsApp Integration Research

    Bot is Telegram-only via grammy. Minimal abstraction (TelegramClient interface).

    Options
    OptionApproachTrade-offs
    AWhatsApp Business Cloud API (Meta)Free user-initiated (24h). Requires FB Business account.
    BVia Twilio / MessageBirdFriendlier API. Twilio partially integrated. Extra cost.
    Cwhatsapp-web.js / BaileysFree but ban risk. Not for production.
    Required refactoring
    • TelegramClient → MessagingPlatform interface
    • Handler adapter layer (grammy → platform-agnostic)
    • Media handling, session keys, Telegram-specific features
    ▶ Discord Integration Deferred

    Blocked by WhatsApp / MessagingPlatform abstraction. Library: discord.js.
